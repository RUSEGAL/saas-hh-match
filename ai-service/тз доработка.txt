ТЗ: Доработка AI Service — Vacancy Matching
1. Overview
Добавить функционал поиска и сопоставления вакансий hh.ru с резюме пользователя.
Flow:
User → POST /api/vacancies/match → NATS (vacancy.match) → AI Service
                                                         ↓
                                              1. Fetch resume from API
                                              2. Parse hh.ru vacancies
                                              3. AI semantic matching
                                                         ↓
User ← POST /ai/webhook/matches ←─────────────────────────
2. New NATS Queue
Queue: vacancy.match
Job structure:
type VacancyMatchJob struct {
    ResumeID int64  `json:"resume_id"`
    UserID   int64  `json:"user_id"`
    Query    string `json:"query"` // e.g., "golang developer"
    Limit    int    `json:"limit"` // max vacancies to check (default: 20)
}
3. New Endpoints in main API
3.1 GET /api/resumes/:id (internal, for AI Service)
Get resume content by ID (AI Service needs this).
Response:
{
  "id": 1,
  "title": "Backend Developer",
  "content": "Full resume text...",
  "tags": ["go", "postgres"]
}
3.2 POST /api/vacancies/match
Request matching vacancies for a resume.
Request:
{
  "resume_id": 1,
  "query": "golang developer",
  "limit": 20
}
Response:
{
  "job_id": "uuid",
  "status": "processing"
}
3.3 GET /api/vacancies/matches/:job_id (optional)
Check matching status/results.
4. New Webhook: matches
Endpoint: POST /ai/webhook/matches
Request from AI Service:
{
  "user_id": 1,
  "resume_id": 1,
  "query": "golang developer",
  "matches": [
    {
      "vacancy_title": "Senior Go Developer",

      "salary": "от 200 000 ₽",
      "excerpt": "Краткое описание вакансии..."
    }
  ]
}
5. Database
5.1 New table: vacancy_matches
CREATE TABLE vacancy_matches (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    resume_id INTEGER NOT NULL,
    query TEXT NOT NULL,
    status TEXT DEFAULT 'pending', -- pending, completed
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY(user_id) REFERENCES users(id) ON DELETE CASCADE
);
5.2 New table: vacancy_match_results
CREATE TABLE vacancy_match_results (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    match_id INTEGER NOT NULL,
    vacancy_title TEXT,
    vacancy_company TEXT,
    score REAL,
    url TEXT,
    salary TEXT,
    excerpt TEXT,
    FOREIGN KEY(match_id) REFERENCES vacancy_matches(id) ON DELETE CASCADE
);
6. AI Service Logic
6.1 Vacancy Fetcher
Use hh.ru API:
GET https://api.hh.ru/vacancies?text={query}&per_page={limit}
Parse fields:
- name (title)
- employer.name (company)
- salary (if present)
- alternate_url (vacancy URL)
- snippet.preview_text (excerpt)
6.2 Resume Fetcher
Call main API:
GET http://main-api:8080/internal/resumes/{id}
6.3 Matching Algorithm
For each vacancy:
1. Combine vacancy title + excerpt
2. Send to AI for semantic comparison
Prompt:
Compare this resume with the vacancy. Return match score 0.0-1.0.
Resume:
{resume_content}
Vacancy:
{vacancy_title} at {company}
{excerpt}
Return JSON:
{
  "score": 0.0-1.0,
  "reasoning": "brief explanation"
}
6.4 Filter
Return only vacancies with score >= 0.70.
