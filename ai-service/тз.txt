Техническое задание: AI Resume Analyzer Service
1. Overview
Микросервис для AI-анализа и оптимизации резюме.
Flow:
Go API → NATS (resume.analyze) → AI Service → Webhook → Go API (updates DB)
2. Tech Stack
Component	Technology
Language	Go
Queue	NATS
HTTP Client	net/http или resty
AI	OpenAI API (GPT-4) или локальная модель
Config	viper / env
3. Functions
3.1 NATS Consumer
- Подключение к NATS (NATS_URL env)
- Подписка на resume.analyze
- Парсинг ResumeAnalysisJob:
    type ResumeAnalysisJob struct {
      ResumeID int64
      UserID   int64
      Title    string
  }
  
3.2 AI Analysis
- Отправка title в GPT-4 с промптом на анализ
- Получение оптимизированного контента
- Извлечение тегов (skills, technologies)
- Расчёт score (0.0 - 1.0)
Prompt пример:
Analyze this resume title and optimize it.
Title: {title}
Return JSON:
{
  "optimized_title": "...",
  "optimized_content": "...",
  "tags": ["skill1", "skill2"],
  "score": 0.0-1.0
}
3.3 Webhook Sender
- POST http://api:8080/ai/webhook/analyze
- Retry логика (3 попытки с backoff)
- Логирование результата
Request:
{
  "resume_id": 1,
  "title": "Optimized Backend Developer",
  "content": "...",
  "tags": ["go", "postgres", "microservices"],
  "score": 0.85
}
4. Config
NATS_URL=nats://localhost:4222
API_WEBHOOK_URL=http://localhost:8080/ai/webhook/analyze
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4

6. Error Handling
Scenario	Action
NATS disconnect	Reconnect с backoff
AI API error	Retry 3x, затем log + skip
Webhook fail	Retry 3x с exponential backoff
Invalid job	Log + ack (don't requeue)
7. Logging
- Использовать zerolog
- Логировать: job received, analysis result, webhook success/fail
- Структурный формат (JSON)
8. Docker
FROM golang:1.26-alpine
WORKDIR /app
COPY . .
RUN go build -o ai-service ./cmd/main.go
CMD ["./ai-service"]
